Overview

Product

Private static hosting

Push a file or folder, receive a private URL, and serve it behind the same OIDC trust boundary as the management API.

Runtime

Single Go server

The server is stateless on local disk. Object storage owns blobs, manifests, current pointers, and trash.

Distribution

CLI-first workflow

Release binaries are published to GitHub Releases and installed through the custom Homebrew tap.

Publish Workflow

1. Login

CLI discovers server auth config and completes OIDC PKCE login.

2. Hash

CLI walks files, normalizes paths, computes SHA-256 hashes, and prepares manifest metadata.

3. Check

Server reports missing hashes and returns signed PUT URLs for missing blobs.

4. Upload

CLI uploads missing blobs directly to object storage through signed URLs.

5. Commit

Server stores the manifest and atomically updates the slug current pointer.

Storage And Data Model

Bucket layout
  • blobs/sha256/<hash>
  • manifests/<slug>/<id>.json
  • slugs/<slug>/current
  • _trash/<hash>
Limits
  • 100 files per push
  • 1 GiB per file
  • 3 GiB per push
  • 10 manifests retained per slug
Backends
  • GCS through gs://... Go CDK URLs
  • IAM signBlob for GCS signed URLs
  • S3-compatible explicit config for Garage/R2

Auth Model

CLI

Uses OIDC loopback PKCE. Tokens are stored per server in the OS keychain with a file fallback. Google deployments use a Desktop app OAuth client for CLI login.

Browser

Uses a web OAuth client and signed jot_session cookie. Claims are checked on every authenticated request.

Authorize rules

Authorization can match required claims such as hosted domain, or allowed email domains. Production policy is configured outside public repo files.
